

# 基于反射

gbox的奠基之一就是用反射来做链式调用，目的是使开发者的逻辑清晰
争议中，可能在实际使用中不太重要


# 序列化

	json 或 proto 的 marshal相关操作，没什么争议

# 消息机制

	把一切通信都抽象成 消息 的概念，先定义基础的报文体，然后在上层封装不同协议（http, ws, grpc等）的交换，甚至可以延伸到落地、缓存等，通过标准的消息机制，开发者甚至不用管理每一层的数据交换逻辑

# 持久化

	1. 标准化数据字段的定义
	2. 理想中的dao可以做成独立的服务，它包含redis防穿透、同步锁、定时落地等基建保障，如果有增删改的操作还要上锁和csrf-token
	3. 标准化的查询指令，如果深度使用AI，应该可以用一套标准的QL去查询几乎所有的数据库
